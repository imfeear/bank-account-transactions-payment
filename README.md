# JalaBank Project

This repository contains the source code for the JalaBank project, a web development project simulating a banking system.

## Modules

The project is divided into the following modules:

- **Accounts and Transactions**: Handles account creation, balance inquiries, deposits, withdrawals, transfers, and statement generation.
- **Loans**: Manages loan applications, simulations, payment plans, and loan monitoring.
- **Payments**: Processes bill payments, manual payments, payment scheduling, and payment history.
- **Credit Card**: Provides features for card management, including password management, cancellation/blocking, usage notifications, limit management, invoice viewing and payment, and purchase history.

## Requirements Analysis

Detailed requirements analysis for each module can be found in the respective module directories. A brief overview is provided below:

### Accounts and Transactions

- **Functional Requirements**: Account opening, balance inquiry, transactions (deposit, withdrawal, transfer), statement generation, authentication, authorization, user data update, notifications.
- **Non-Functional Requirements**: Security, performance, scalability, reliability, usability, maintainability, compatibility.

### Credit Card

- **Functional Requirements**: Card password management, cancellation/blocking, usage notification, limit management, card invoice viewing and payment, card issuance request and validation process, purchase history, integration with account systems.
- **Non-Functional Requirements**: Security, performance, availability, and efficiency for each functionality.

### Loans

- **Functional Requirements**: Consult customer data, customer profile evaluation, loan simulation, payment plan generation, loan simulation summary, user authentication, contract acceptance, loan amount deposit, payment installment creation, loan storage, credit reassessment, loan monitoring.
- **Non-Functional Requirements**: Integration with account and transaction modules, database communication, performance, usability.

### Payments

- **Functional Requirements**: Bill payment, manual payments, payment scheduling, payment history.
- **Non-Functional Requirements**: Security, performance, reliability, scalability, compliance.
